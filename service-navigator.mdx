## Overview

The Service Navigator provides step-by-step guidance for 117 Bulgarian government e-services across 14 categories, helping citizens understand requirements and complete applications.

## Features

- **117 services** across 14 categories
- **Step-by-step guides** with detailed instructions
- **Document checklists** for each service
- **Official links** to government portals
- **Search and filtering** by category and keywords
- **Mobile-responsive** design

## Service categories

1. **Civil services** - Birth certificates, ID cards, passports
2. **Business services** - Company registration, licenses
3. **Traffic services** - Driver's licenses, vehicle registration
4. **Healthcare services** - Medical records, prescriptions
5. **Education services** - School enrollment, diplomas
6. **Social services** - Benefits, pensions
7. **Tax services** - Tax declarations, payments
8. **Property services** - Property registration, permits
9. **Legal services** - Court documents, notary services
10. **Employment services** - Job registration, unemployment benefits
11. **Environmental services** - Permits, inspections
12. **Cultural services** - Library cards, museum passes
13. **Agricultural services** - Subsidies, permits
14. **Other services** - Miscellaneous government services

## Service structure

Each service includes:

```typescript
interface Service {
  id: string
  slug: string
  title: string
  summary: string
  category: string
  agency: string
  steps: Step[]
  links: Link[]
  documents?: string[]
  prerequisites?: string[]
  fees?: Fee[]
  processingTime?: string
}

interface Step {
  id: string
  title: string
  description: string
  documents?: string[]
  tips?: string[]
}

interface Link {
  label: string
  href: string
}

interface Fee {
  description: string
  amount: number
  currency: string
}
```

## API endpoints

### GET /api/services

Get all services with optional filtering.

**Query parameters:**
- `category` - Filter by category
- `search` - Search query
- `limit` - Results per page (default: 20)
- `offset` - Pagination offset

**Example:**
```bash
curl "http://localhost:3000/api/services?category=civil&limit=10"
```

### GET /api/services/[id]

Get service details by ID.

**Example:**
```bash
curl "http://localhost:3000/api/services/passport-application"
```

### GET /api/services/categories

Get all categories with service counts.

**Example:**
```bash
curl "http://localhost:3000/api/services/categories"
```

## Search and filtering

### Client-side search

```typescript
// lib/services/service-search.ts
import Fuse from "fuse.js"

const fuse = new Fuse(allServices, {
  keys: ["title", "summary", "category"],
  threshold: 0.3
})

export function searchServices(query: string): Service[] {
  if (!query) return allServices
  
  return fuse.search(query).map(result => result.item)
}
```

### Category filtering

```typescript
export function filterByCategory(
  services: Service[],
  category: string
): Service[] {
  if (!category) return services
  
  return services.filter(s => s.category === category)
}
```

## Integration with government portals

### Portal.IS integration

```typescript
// lib/egov/providers/portal-is.ts
export class PortalISProvider {
  async submitApplication(
    serviceId: string,
    data: Record<string, any>
  ): Promise<ApplicationResult> {
    const response = await fetch(
      `${process.env.PORTAL_IS_API_URL}/applications`,
      {
        method: "POST",
        headers: {
          "Authorization": `Bearer ${process.env.PORTAL_IS_TOKEN}`,
          "Content-Type": "application/json"
        },
        body: JSON.stringify({
          serviceId,
          data
        })
      }
    )
    
    return await response.json()
  }
}
```

## Analytics

Track service usage:

```typescript
// lib/services/analytics.ts
export async function trackServiceView(serviceId: string) {
  await prisma.serviceView.create({
    data: {
      serviceId,
      timestamp: new Date()
    }
  })
}

export async function getPopularServices(limit: number = 10) {
  return await prisma.serviceView.groupBy({
    by: ["serviceId"],
    _count: true,
    orderBy: {
      _count: {
        serviceId: "desc"
      }
    },
    take: limit
  })
}
```

## Troubleshooting

### Service not found

- Check service slug is correct
- Verify service is exported in `services.ts`
- Clear Next.js cache: `rm -rf .next`

### Application submission fails

- Check provider configuration
- Verify API credentials
- Check network connectivity
- Review error logs

### Search not working

- Rebuild search index
- Check Fuse.js configuration
- Verify service data structure
